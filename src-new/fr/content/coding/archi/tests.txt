===================
Tests programmables
===================

Les tests programmables se feront dans le dossier dédié path::''tests'' hors du dossier path::''src''
((
    On veut se mettre dans une situation réelle d'utilisation.
)).
La construction de ces tests utilisera les principes suivants.

    1) Utilisation du retour d'expérience.

    1) Construction de tests fonctionnels sans connaître les arcanes du code \src.

    1) Comme indiqué dans cf::''tools'', intégration des exemples utilisés dans les \docs "utilisateur" et \techs.

    1) Utilisation de tests intégrés dans les contributions peu \techs.
