===================================
Fichiers pour la \doc "utilisateur"
===================================

Le codeur se charge de rédiger une \doc "utilisateur" dans la langue indiquée via le fichier path::''about.yaml'' spécifique au projet
((
    Voir cf::''?/metainfos/allpacks.txt''.
)).
Ceci se fait au même niveau que les fichiers ext::''sty'' : pour cela, il suffit de rédiger des fichiers ext::''tex'' en respectant des règles de nommage que nous allons expliquer via deux cas d'utilisation possibles.


Pour commencer, considérons le dossier suivant où un fichier path::''about.yaml'' permet de choisir l'ordre d'utilisation des fichiers.

dirtree::
    + poor-lonesome-cowboy
        * about.yaml
        * lucky-1.tex
        * lucky-2.tex
        * lucky.sty
        * rantanplan.tex
        * rantanplan.sty


Le fichier path::''about.yaml'' admet le contenu suivant afin de considérer le fichier path::''rantanplan.sty'' avant path::''lucky.sty''.

yaml::
    ---
    title = Le fichier path::''poor-lonesome-cowboy/about.yaml''
    ---

    toc:
      - rantanplan.sty
      - lucky.sty


Aucune \ref n'étant faite aux fichiers ext::''tex'', les règles suivantes sont appliquées.

    1) Les fichiers ext::''tex'' commençant par path::''rantanplan-'' \etou le fichier path::''rantanplan.tex'' forment une liste qui sera ordonnée de façon \lexico pour l'intégration des fichiers ext::''tex'' dans la \doc finale.

    1) Viennent ensuite les fichiers commençant par path::''lucky-'' \etou le fichier path::''lucky.tex'' ordonnés de façon \lexico pour l'intégration dans la \doc finale.


note::
    Rien n'empêche d'indiquer à la fois des fichiers ext::''sty'' et ext::''tex'' dans un ordre souhaité. Ceci peut s'avérer utile pour un document ext::''tex'' non associé à un code ext::''sty'' mais présentant rapidement différentes fonctionnalités ; un tel fichier devant être traité avant tout le reste, il suffit de le mettre au début de la liste yaml::''toc'' du fichier path::''about.yaml''.


Des règles similaires s'appliquent au dossier sans fichier path::''about.yaml'' comme ci-après, un cas pour lequel l'intégration dans la \doc se fera avec path::''alpha.tex'' suivi de path::''omega3-a.tex'' puis de path::''omega3-b.tex''.

dirtree::
    + alpha-et-omega3
        * alpha.tex
        * alpha.sty
        * omega3-a.tex
        * omega3-b.tex
        * omega3.sty
