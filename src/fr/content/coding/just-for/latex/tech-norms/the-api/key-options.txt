=======================
Des options de type clé
=======================

note::
    En coulisse, les options de type clé sont codées à la sauce \l3.


Afin d'éviter les conflits de noms dans le code \src, un groupe de clé est nommé via un chemin virtuel de l'un des types suivants.

    1) ''project-name / source-file-name''

    1) ''project-name / source-file-name / macro-env-name''


Voici un exemple de code \l3 présent dans un fichier \src path::''main.sty'' relatif à un \pack verb::''fictpack''. Comme nous définissons ici des clés pour un réglage global du package, nous utilisons un chemin virtuel court.

l3::
    \keys_define:nn { fictpack / main } {
      theme .choices:nn = {
        color,
        black-white,
        white-black
      } {
        \tl_set:Nn \l_fictpack_theme_str
          { \tl_use:N \l_keys_choice_tl }
      },
      theme .initial:n = color,
      theme / unknown .code:n =
        \msg_error:nnxxx { fictpack ~ (srcfile) }
          { package-option-unknown-theme }
    }

    \ProcessKeyOptions[fictpack]
